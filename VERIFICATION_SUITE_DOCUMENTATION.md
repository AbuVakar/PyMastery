# PyMastery Automated Verification Suite Documentation

## 🎯 Overview

The PyMastery Automated Verification Suite provides comprehensive testing for authentication, dashboard, AI tutor, and route integrity. This suite ensures that all critical user flows work correctly and can be verified non-interactively after every code change.

## 🚀 Running Tests

### Quick Verification (Fast)
```bash
python quick_verification.py
```
Runs basic health checks and build verification in under 2 minutes.

### User Flow Tests (Medium)
```bash
python test_user_flows.py
```
Tests complete user journeys from signup to AI tutor interaction.

### Comprehensive Verification (Complete)
```bash
python run_verification.py
```
Runs all test suites including backend, frontend, and integration tests.

### Individual Test Suites

#### Backend Tests
```bash
cd backend
python -m pytest tests/test_verification_suite.py -v
```

#### Frontend Tests
```bash
cd frontend
npm run test:verification
```

#### Health Checks
```bash
cd backend
python test_health_direct.py
```

#### Security Tests
```bash
cd backend
python test_auth_security.py
```

## 📝 Test Scenarios

### Authentication Flow
1. **New User Registration**
   - Valid data → Success
   - Invalid email → Validation error
   - Weak password → Validation error
   - Duplicate email → Conflict error

2. **User Login**
   - Valid credentials → Success
   - Invalid password → Auth error
   - Non-existent user → Auth error

3. **Token Management**
   - Token validation → Success
   - Token refresh → New token
   - Invalid token → Auth error
   - Missing token → Auth error
   - Logout → Token invalidation

### Dashboard Flow
1. **Data Retrieval**
   - Authenticated user → Stats data
   - Unauthenticated user → Auth error
   - Invalid token → Auth error

2. **Data Validation**
   - Required fields present
   - Data types correct
   - Response format consistent

### AI Tutor Flow
1. **Message Processing**
   - Valid question → Response
   - Empty message → Validation error
   - Whitespace only → Validation error
   - Long message → Response

2. **Error Handling**
   - Service unavailable → Error response
   - Timeout → Error response
   - Invalid format → Error response

## 🔄 Integration with CI/CD

### GitHub Actions Example
```yaml
name: Verification Tests
on: [push, pull_request]

jobs:
  verify:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Setup Python
        uses: actions/setup-python@v2
        with:
          python-version: '3.9'
      
      - name: Setup Node.js
        uses: actions/setup-node@v2
        with:
          node-version: '16'
      
      - name: Start Backend
        run: |
          cd backend
          python -m uvicorn main:app --reload &
          sleep 10
      
      - name: Run Verification
        run: python run_verification.py
      
      - name: Upload Results
        uses: actions/upload-artifact@v2
        with:
          name: verification-results
          path: verification_results.json
```

## 🚨 Troubleshooting

### Common Issues

#### Backend Not Running
```
❌ Backend Health: Connection refused
```
**Solution:** Start the backend server
```bash
cd backend
python -m uvicorn main:app --reload
```

#### Frontend Build Fails
```
❌ Frontend Build: Failed
```
**Solution:** Install dependencies
```bash
cd frontend
npm install
```

#### Test Timeouts
```
❌ Test Failed: Timeout
```
**Solution:** Increase timeout or check performance
```bash
export TEST_TIMEOUT=30
```

#### Import Errors
```
❌ ModuleNotFoundError: No module named 'pytest'
```
**Solution:** Install test dependencies
```bash
cd backend
pip install pytest pytest-json-report httpx
```
